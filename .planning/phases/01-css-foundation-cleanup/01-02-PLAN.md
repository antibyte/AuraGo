<task type="auto">
  <name>Task 2: Fix .cfg-input-flex and .cfg-password-input min-width</name>
  <files>ui/css/config.css</files>
  <action>
Read config.css to find .cfg-input-flex and .cfg-password-input rules.

Replace fixed min-width:240px with fluid equivalents:

.cfg-input-flex { flex:1; min-width:240px; }
  ->
.cfg-input-flex { flex:1; min-width:min(240px, 50%); }

Or better: .cfg-input-flex { flex:1; min-width:0; } since it is inside a flex container and flex:1 already provides the grow behavior.

.cfg-password-input { flex:1; min-width:240px; }
  ->
.cfg-password-input { flex:1; min-width:min(240px, 50%); }
  or .cfg-password-input { flex:1; min-width:0; }

Use min-width:0 as the primary fix since these are flex children and 0 is the natural minimum for flex items. The 240px was forcing overflow on small viewports.

Also scan for any other hardcoded min-width values > 200px that are likely to cause overflow (grep results showed 3327, 5113, 5502 as potentially problematic). Only fix ones that are explicitly causing overflow issues -- do NOT arbitrarily change working min-width values.
  </action>
  <verify>
    <automated>grep -n "min-width:240px" ui/css/config.css | wc -l</automated>
  </verify>
  <done>No .cfg-input-flex or .cfg-password-input rules with min-width:240px remain</done>
</task>
