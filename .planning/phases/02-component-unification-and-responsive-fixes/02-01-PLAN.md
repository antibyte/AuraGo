## Decisions to honor (from 02-CONTEXT.md):

**D-07:** Fix pills overflow — use `overflow-wrap: break-word` and `min-width: 0` on pill containers
**D-08:** Fix title truncation — ensure `.mission-name` has `overflow: hidden`, `text-overflow: ellipsis` with proper `min-width: 0` on flex parent
**D-09:** Fix button cutoff — use `flex-shrink: 0` on action buttons, ensure `.mission-actions` doesn't overflow card width
**D-10:** Badge consistency — Mission Control badges should use same `.aura-badge` classes as Dashboard

## Target CSS selectors in missions.css:

- `.mission-card` — card container
- `.mission-pills` or `.mission-badges` — pill/badge container (needs overflow-wrap, min-width: 0)
- `.mission-name` — title element (needs overflow hidden, ellipsis, min-width: 0 on parent flex)
- `.mission-actions` — action button container (needs flex-shrink: 0 on buttons)
</context>

<task type="auto">
  <name>Task 2: Fix mission name title truncation</name>
  <files>ui/css/missions.css</files>
  <action>
Fix title truncation per D-08. Ensure `.mission-name` (or equivalent heading selector) has proper overflow handling:

```css
/* D-08: Prevent title truncation — needs min-width: 0 on flex parent too */
.mission-header {
  min-width: 0; /* flex parent must allow shrinking */
}

.mission-name {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap; /* or remove if wrapping preferred */
}
```

If `.mission-name` is a direct child of a flex container, the flex container must have `min-width: 0` to prevent the flex item from forcing overflow. If `.mission-name` is inside `.mission-header`, apply `min-width: 0` to `.mission-header`.

Alternatively, if wrapping titles are preferred:
```css
.mission-name {
  overflow-wrap: break-word;
  white-space: normal;
}
```

Choose the pattern that best matches the existing design intent — the key is preventing overflow/clipping.
  </action>
  <verify>
    <automated>grep -n "overflow.*hidden\|text-overflow.*ellipsis\|min-width: 0" ui/css/missions.css | grep -E "mission-name|mission-header"</automated>
  </verify>
  <done>Mission name titles are fully visible — either with ellipsis on single line or wrapping naturally without overflow</done>
</task>
