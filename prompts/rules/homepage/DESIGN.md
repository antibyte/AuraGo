# AuraGo Homepage Design System

## Colors
- **Primary** `#14B8A6`: Main action color for links, focused controls, and active states.
- **Accent** `#38BDF8`: Secondary emphasis for progress, previews, and informational highlights.
- **Background** `#0F172A`: Dark application background for technical tools and dashboards.
- **Surface** `#172033`: Main panel and section surface.
- **Surface Muted** `#1E293B`: Inputs, secondary panels, and table rows.
- **Text Primary** `#E5EEF7`: Main readable text on dark surfaces.
- **Text Secondary** `#94A3B8`: Labels, helper text, timestamps, and metadata.
- **Success** `#22C55E`, **Warning** `#F59E0B`, **Danger** `#EF4444`: Semantic states only.

## Typography
- Use Inter, system UI, or the project font stack.
- Body text: 16px, regular, line-height 1.5.
- Compact UI text: 14px, medium, line-height 1.35.
- Page headings: 28-40px, 700 weight, line-height 1.1.
- Section headings: 18-22px, 700 weight.
- Do not use negative letter spacing.

## Spacing
- Base spacing unit: 4px.
- Small gaps: 8px.
- Standard component padding: 12px to 16px.
- Section spacing: 32px to 48px.
- Keep dense operational surfaces compact but readable.

## Components
- Buttons: 8px radius or less, clear hover/focus states, icon plus text when the command benefits from recognition.
- Inputs: consistent height, visible focus ring, high-contrast placeholder and value text.
- Cards: use only for repeated items, modal content, or framed tools; avoid cards inside cards.
- Tables and lists: stable columns, readable row height, clear status badges, no overlapping text.
- Navigation: predictable tabs or side navigation for tools; avoid marketing-style navigation in operational apps.

## Elevation
- Prefer subtle borders and surface contrast over heavy shadows.
- Use shadows only for modals, menus, and floating overlays.
- Avoid decorative orbs, bokeh blobs, and one-note gradient backgrounds.

## Guidelines
- Build the actual usable experience as the first screen, not a marketing explanation.
- Use real or generated bitmap imagery when a website needs visual assets.
- Keep color palettes balanced; avoid UIs dominated by a single hue family.
- Ensure all text fits inside controls at mobile and desktop sizes.
- Verify desktop and mobile viewports before declaring visual work complete.
