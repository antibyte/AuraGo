ISSUE 4: PLACEHOLDER MISMATCH IN TRANSLATIONS
         (Non-English files have different brace style than English)
================================================================================

  Because en.json uses single-brace {count} for the key
  "dashboard.knowledge_quality_duplicate_count", while 4 translations
  correctly use double-brace {{count}} (matching project convention),
  there is a structural mismatch:

  English:       "{count} Nodes"          (single brace - BUG in en.json)
  Japanese:      "{{count}} Nodes"         (double brace - correct)
  Norwegian:     "{{count}} noder"         (double brace - correct)
  Polish:        "{{count}} wezlow"        (double brace - correct)
  Portuguese:    "{{count}} nos"           (double brace - correct)

  Affected files:
    ui/lang/dashboard/ja.json:512
    ui/lang/dashboard/no.json:512
    ui/lang/dashboard/pl.json:512
    ui/lang/dashboard/pt.json:512

  Root cause: en.json line 512 should be "{{count}} Nodes" not "{count} Nodes".
  The 4 translation files are actually correct; the English file is wrong.
